📁 .github/
👉 À quoi ça sert

Contient tout ce qui est lié à GitHub, surtout l’automatisation.

Qui l’utilise

GitHub

Toi, indirectement

Contenu typique
.github/
└── workflows/
    └── tests.yml

Exemple concret

lancer des tests automatiquement à chaque git push

vérifier que ton code ne casse rien

👉 Optionnel au début, utile plus tard.

📁 docs/
👉 À quoi ça sert

Documentation détaillée (au-delà du README).

Qui l’utilise

Utilisateurs avancés

Contributeurs

Contenu typique
docs/
├── installation.md
├── usage.md
├── troubleshooting.md


👉 Si ton README suffit → tu peux t’en passer.

📁 src/ (LE PLUS IMPORTANT)
👉 À quoi ça sert

Contient le code principal de WindowsToolKit.

Qui l’utilise

PowerShell (via Import-Module)

Ton script Main.ps1

Ce qu’on y met
src/
├── WindowsToolKit.psm1   ← module principal
├── WindowsToolKit.psd1   ← manifest (version, auteur…)
├── Public/               ← commandes utilisateur
└── Private/              ← code interne

👉 Indispensable

📁 src/Public/
👉 À quoi ça sert

Fonctions accessibles à l’utilisateur.

Exemple
Get-WtkSystemInfo.ps1
Start-Wtk.ps1


👉 Ce sont les commandes que tu veux exposer.

📁 src/Private/
👉 À quoi ça sert

Fonctions internes, invisibles pour l’utilisateur.

Exemple
Show-Menu.ps1
Write-InternalLog.ps1


👉 Ça évite le bazar et protège ton API.

📁 lib/
👉 À quoi ça sert

Code de support ou dépendances, pas spécifiques à WTK.

Qui l’utilise

Le module (src)

Pas l’utilisateur directement

Exemples
lib/
├── Logger/
├── ConsoleUI/
└── ZipHelpers/


👉 Optionnel
➡️ À créer seulement si tu as du code réutilisable
